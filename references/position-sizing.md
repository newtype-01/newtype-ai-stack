# Position Sizing

用于把 AI 产业链候选公司分为 Core 7、N-Stack 20、Watch List 50,并给出研究框架下的目标权重、调仓信号和风险控制。

## 默认组合架构

- **Core 7**: 约 7 家核心持仓,占 AI 相关仓位 60-75%。
- **N-Stack 20**: 约 20 家扩展配置,占 20-30%。
- **Watch List 50**: 观察池或机会性小仓,占 0-10%。
- **现金/非 AI**: 默认 5-15%;若用户未给风险偏好,默认中性偏激进,总 AI 敞口 75%。

不需要凑满数量。候选中只有 4 家符合 Core 标准,Core 就 4 家。

## 输入

- 候选公司清单: 通常 5-15 个 ticker。
- 可选: 风险偏好、现有持仓、总资金规模、税务或地区限制。

未给现有持仓时,输出目标权重而非交易指令。

## 分层标准

### Core 7

尽量满足:

- 位于第1-4层,或第5层中具有强卖铲子属性。
- 终局高概率或终局候选。
- 卖铲子属性强。
- 核心稀缺或次级稀缺。
- 能持有 18 个月以上且核心论点稳定。

典型单家公司权重: 8-15%;单一公司上限 25%。

### N-Stack 20

满足 3 项以上:

- 第1-4层优先,第5层可选。
- 终局候选、不确定,或当前/下一瓶颈的高质量过渡期赢家。
- 当前主瓶颈次级玩家,或下一个瓶颈提前布局对象。
- 有明确催化: 财报、订单、产品发布、客户验证。
- 估值未充分反映稀缺性。

典型单家公司权重: 1-3%。

### Watch List 50

满足 2 项以上:

- 与 AI 产业链相关。
- 有特定催化等待触发。
- 估值或事件有吸引力。
- 不确定性高但弹性大。

典型权重: 0-1%;多数只跟踪不持仓。

## 五个分仓原则

1. **风险预算先行**
   - 激进: AI 敞口 85-95%。
   - 中性: AI 敞口 60-80%。
   - 保守: AI 敞口 30-50%。

2. **层级分散 + 单点重仓**
   - Core 尽量跨至少 4 层,N-Stack 尽量跨至少 5 层。

3. **卖铲子优先**
   - 组合中卖铲子公司目标占 70%+;挖金子公司不超过 20%。

4. **终局赢家压舱**
   - Core 中至少 3 家应是终局高概率或强终局候选。

5. **单一公司风险上限**
   - 单家公司不超过 25%,即使是最强核心标的。

## 工作流

1. **逐一打标签**
   - 七层定位: 必要时用 `stack-mapping.md`。
   - 稀缺性: 核心 / 次级 / 可替代。
   - 终局判断: 必要时用 `bridge-vs-endgame.md`。
   - 卖铲子属性: 是 / 否 / 介于之间。

2. **分入 Core / N-Stack / Watch**
   - 不硬凑数量。
   - 对缺乏信息的公司先放 Watch,列验证项。

3. **检查分散度和风险**
   - 层级集中、客户集中、地缘政治、利率、AI 周期回撤。

4. **计算目标权重**
   - 先确定总 AI 敞口。
   - 再分配 Core / N-Stack / Watch / 现金。
   - 单家公司权重与终局确定性、稀缺性、催化可验证性匹配。

5. **定义调仓信号**
   - 升级: 连续财报验证、大客户订单、技术节点、产业瓶颈增强、估值回合理区间。
   - 降级: 连续低于预期、大客户流失、路线被替代、稀缺性消失、泡沫化。

## 质量底线

- 不为凑满 Core 7 / N-Stack 20 而强行分配。
- 权重必须服务于风险预算,不是服务于喜好。
- 明确区分目标配置、现有持仓调整和具体交易指令。
- 对估值、财报、价格、份额等当前数据必须要求实时验证。
