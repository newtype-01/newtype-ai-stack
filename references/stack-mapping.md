# Stack Mapping

用于把公司、ticker、产品、技术、概念或产业事件定位到 newtype AI Stack 七层产业地图中。

## 输入

用户可提供:

- 公司或 ticker: "分析 VRT"、"ASML 在哪一层"
- 产品或技术: "CPO 光模块"、"Mojo 编程语言"
- 新闻或事件: "OpenAI 发布 Sora 2 属于哪一层事件"
- 概念: "Agent 框架"、"向量数据库"

只给公司名时,默认分析其主要利润来源;如跨业务线,说明核心利润来源和其他层级延伸。

## 工作流

1. **识别业务/技术本质**
   - 它实际卖什么?
   - 客户是谁?
   - 直接上游供应商是谁?
   - 主要利润来源是什么?

2. **七层归类**
   - 依据"卖什么 + 卖给谁"归到一个核心层。
   - 跨层公司先按核心利润来源归类,再说明其他层级。
   - 技术服务多层时,按直接服务的客户/环节归类。

3. **计算距离终端算力需求**
   - 使用 `shared-framework.md` 的距离定义。
   - 说明传导链: 它服务谁,那一层再服务谁。

4. **应用卖铲子定律**
   - 判断供给侧/应用侧/介于两者之间。
   - 说明如果 AI 需求继续增长,它如何受益;如果应用赢家未定,它是否仍能赚钱。

5. **评估稀缺性**
   - 核心稀缺 / 次级稀缺 / 可替代。
   - 必须给出可替代玩家数量或替代难度,不能只说"高/中/低"。

6. **给出跟踪建议**
   - 核心跟踪 / 战术跟踪 / 观察池 / 过滤掉。
   - 附后续验证清单。

## 输出模板

```markdown
## 结论先行

**[公司/技术/事件] 定位**: 第 N 层 [层级名]
**距离终端算力需求**: 0 / 1 / 2 / 3
**卖铲子 vs 挖金子**: [卖铲子 / 挖金子 / 介于之间]
**稀缺性等级**: 核心稀缺 / 次级稀缺 / 可替代
**跟踪建议**: 核心跟踪 / 战术跟踪 / 观察池 / 过滤掉

一句话总结其在 newtype AI Stack 里的角色。

## A. 业务/技术本质
它实际上在卖什么?客户是谁?主要利润来源是什么?

## B. 七层定位
为什么归到第 N 层?如果跨层,说明核心利润来源和其他延伸。

## C. 上下游关系
- 上游供应商:
- 下游客户:
- 产业链位置:

## D. 距离终端算力需求
距离 = N,并说明传导链。

## E. 卖铲子定律检验
它如何从 AI 需求增长中受益?是否依赖某个应用赢家?

## F. 稀缺性评估
全球玩家数量、替代难度、毛利率稳定性。

## G. 跟踪建议
是否适合 Core 7 / N-Stack / Watch List / 过滤。

## H. 关键风险
技术路线变化、业务转型、客户集中、归类争议。

## I. 后续验证清单
财报、订单、价格、份额、客户、产能等要验证的指标。
```

## 质量底线

- 看实际业务和客户结构,不要被公司名字、营销话术、概念热度带偏。
- 一家公司只能有一个核心层级;跨层必须解释利润占比或战略延伸。
- 不要把第7层应用公司硬抬为 AI 核心受益者。
- 稀缺性判断要落到具体玩家数量、替代周期或客户切换成本。
